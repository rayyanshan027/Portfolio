# Cyan

## Summary

Cyan is a distributed social publishing app built with Django and React. It combines a feed-first social interface with longer-form publishing, author profiles, post visibility controls, engagement features, and federation-oriented APIs.

## Role

This was a university team project with later portfolio-focused cleanup and presentation work. My contributions focused on frontend redesign, UI consistency, project polish, and integration cleanup across the Django + React app.

## Contributions

- Redesigned the frontend into a responsive feed-first social layout.
- Added cyan/blue branding and a light/dark mode system.
- Updated the home feed, profile page, entry detail view, auth screens, modals, and navigation.
- Improved project documentation and presentation for portfolio use.
- Continued cleanup and integration work across the Django backend and React frontend.

## Features

- Public feed and following-based feed.
- Author profiles with display name, bio, GitHub link, and profile image.
- Posts in plain text, markdown, and image formats.
- Public, friends-only, and unlisted visibility controls.
- Likes, comments, and follow relationships.
- Local and remote author discovery through federation-aware author IDs.
- Django REST API, Swagger docs, and Django admin support.

## Tech Stack

- Backend: Django, Django REST Framework
- Frontend: React, Vite, Tailwind CSS
- Auth: DRF token auth and Basic Auth for node-to-node federation
- Database: SQLite locally, Postgres in production
